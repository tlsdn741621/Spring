스프링 프레임워크의 장점 중 하나가,
다른 프레임워크 와 연동이 쉽다.

데이터베이스 연동시, jdbc 이용해서 직접 DAO 만들어서 ,
작업을 했지만,
MyBatis 디비 프레임워크, 이용하고,
부트 JPA 프레임워크를 이용할 예정.

장점,
1) 인터페이스로 설계,
2) 특정의 sql 문장을 따로 분리해서 작업 가능.
3) 단독으로 이용가능, 스프링 MVC 조합해서 작업도 가능.

특징
- PreparedStatement/ResultSet -, 개별적으로 각각 메서드들을 이용해서
작업을 했다면, 알아서 내장된 기능을 이용해서, 코드 길이가 줄어듬.

- Connection/PreparedStatement , 이용시, 수동으로 자원을 반납 해야하지만,
자동으로 close()

- SQL  분리, 별도의 파일 형식으로 작업함.

연동하는 방식.
1) MyBatis  단독으로 사용가능.
2) Mapper 인터페이스 이용, 즉, 느스한 결합을 이용해서, 작업

표현방식,
1) 메소드에, sql 애너테이션을 이용해서 구현,
예)
@Select("select now()")
    String getNow();

2) sql 파일를 따로 분리하는 xml 파일 생성.

주의사항,
1) 매퍼 인터페이스 정의하고, 메소드 선언,
xml 파일 , 매퍼 인터페이스 이름 동일.

2) <select> , <insert> 태그 내부에 id 속성, 메소드의 이름과 동일.

1), 2) -> 깔맞춤 표현.
