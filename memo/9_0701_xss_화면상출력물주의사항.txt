 XSS (Cross-Site Scripting) 개념

 정의
 XSS는 악의적인 사용자가 웹사이트에 자바스크립트 코드를 삽입하여,
 다른 사용자 브라우저에서 해당 코드를 실행시키는 보안 취약점입니다.
 이는 사용자의 쿠키 탈취, 세션 탈취, 피싱 페이지 유도 등 다양한 공격으로 이어질 수 있습니다.

 1. <h2>${message}</h2>
 📌 개념


 **EL(Expression Language)**을 사용하여 message 변수를 직접 출력합니다.


 JSP에서 가장 일반적인 출력 방식입니다.


 ⚠️ 보안 특징


 HTML 태그나 JavaScript 코드가 포함되어 있으면 그대로 출력됩니다.


 즉, message에 <script>alert(1)</script>와 같은 내용이 있으면 스크립트가 실행되어 XSS 공격에 노출됩니다.

 2. <h2><c:out value="${message}"></c:out></h2>
 📌 개념


 JSTL(JavaServer Pages Standard Tag Library)의 <c:out> 태그를 사용하여 안전하게 값을 출력합니다.


 내부적으로 HTML 특수 문자를 이스케이프 처리(escaping) 하여 출력합니다.


 🔐 보안 특징


 <, >, ", ', & 등의 문자가 **HTML 엔티티(&lt;, &gt; 등)**로 변환되어 브라우저에서 코드가 실행되지 않고 텍스트로 출력됩니다.


 XSS 방지에 적합한 출력 방식입니다.
